## Epic B — 외부 IP 접속 허용 + 인증 강제

### B1. 인증/인가 레이어 도입(필수)
- [ ] 익명 접근 전면 차단(로그인 페이지 제외)
- [ ] 세션 기반 인증 또는 JWT(HttpOnly Secure Cookie) 적용
- [ ] 비밀번호 해시(Argon2/Bcrypt) + 최소 복잡도 정책
- [ ] RBAC(최소 Admin/Viewer 2단계) 적용

**완료 기준(DoD)**
- 인증 없이 어떤 API/화면도 접근 불가(401/302 처리)
- 권한 없는 기능 호출 시 403 반환

### B2. 외부 공개 보안 기본선
- [ ] TLS 강제(리버스 프록시: Caddy/Nginx + Let’s Encrypt)
- [ ] 보안 헤더(HSTS, CSP, X-Frame-Options, X-Content-Type-Options)
- [ ] CSRF 방어(토큰/동일사이트 쿠키)
- [ ] 로그인 레이트리밋/브루트포스 차단(예: 5회 실패 잠금, IP 단위 제한)

**완료 기준(DoD)**
- HTTP 접근 시 HTTPS 리다이렉트 100%
- 인증 관련 OWASP Top10 핵심 체크리스트 통과

### B3. 운영 보안/감사
- [ ] 감사 로그: 로그인 성공/실패, 권한거부, 중요 설정 변경
- [ ] 세션 만료/강제 로그아웃(비활성 30분, 절대 12시간)
- [ ] 비밀정보 분리(.env/secret store), 키 회전 절차 문서화
- [ ] Fail2ban/WAF 연동(선택) + 알림

**완료 기준(DoD)**
- 보안 이벤트 추적 가능(사용자/IP/시간/행위)
- 비정상 로그인 시 알림 발생

## 구현 계획 (보안 포함)

## 1) 아키텍처 변경
- WebApp 미들웨어 계층 추가
  - `AuthMiddleware`: 인증 확인
  - `RateLimitMiddleware`: 로그인/API 요청 제한
  - `SecurityHeadersMiddleware`: 기본 헤더 일괄 주입
- Reverse Proxy 계층 표준화 (TLS termination + 접근제어)
- 기존 `/api/*` 엔드포인트를 인증 필요 자원으로 전환

## 2) 단계별 일정 (1 sprint = 2주 기준)

### Week 1
1. 인증 기초 구현
   - 사용자 모델/비밀번호 해시/세션 발급
   - 로그인/로그아웃/세션 검증 API
2. 라우트 보호 + RBAC 1차
3. 반응형 레이아웃 베이스 작업

### Week 2
1. 모바일 UX 디테일 + 핵심 시나리오 개선
2. TLS/보안 헤더/CSRF/레이트리밋 적용
3. 모바일 E2E + 보안 테스트 + 문서화
4. 릴리즈 게이트 리뷰(테스터/아키텍트)

## 보안 테스트 계획
- 정적 점검: 의존성 취약점 스캔, 시크릿 노출 스캔
- 동적 점검: 무인증 접근, 세션 고정, CSRF, 브루트포스 시뮬레이션
- 권한 점검: Viewer로 Admin 기능 호출 차단 확인
- 회귀 점검: 기존 트레이딩/백테스트 핵심 기능 정상 동작 확인
