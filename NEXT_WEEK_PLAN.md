## 보안 설계 요약

### 1) 인증(Authentication)
- 익명 접근 금지: 모든 API/웹 리소스는 인증 필수
- 세션 기반 인증 권고: 서버 저장 세션 + HttpOnly/Secure 쿠키
- 비밀번호 저장: Argon2/Bcrypt 해시 사용

### 2) 인가(Authorization)
- RBAC 적용: Admin(설정/운영) vs Viewer(조회)
- 권한 없는 요청은 403 처리

### 3) 전송/노출 보호
- TLS 필수, HTTP → HTTPS 강제 리다이렉트
- 민감 정보는 평문 로그 금지, 시크릿은 환경변수/분리 저장

### 4) 웹 보안 통제
- CSRF 방어 토큰
- 로그인 레이트리밋(예: 5회 실패 시 잠금)
- 보안 헤더 표준 적용

### 5) 감사/운영
- 보안 이벤트 감사로그(사용자/IP/시간/행위)
- 비정상 로그인/과다 실패 알림
- 세션 만료 정책(예: 비활성 30분, 절대 12시간)
